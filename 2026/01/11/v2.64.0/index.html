Carpet AMS Addition v26.1

 2026/01/11

新规则

  • 防止服务器暂停（preventServerPause）
    防止服务器60s后暂停

  • 仙人掌扳手拓展（flippinCactusExtras）
    使flippin仙人掌规则能适用于更多方块
    受影响的方块：木桶、雕文书架、合成器、展示架

  • 实验性矿车速度（experimentalMinecartSpeed）
    设置实验性矿车的速度
    启用该规则将覆盖游戏规则

  • 启用实验性矿车（experimentalMinecartEnabled）
    启用实验性矿车
    服务器环境切换该规则设置时需要重启服务器
    客户端环境切换该规则设置时需要重启客户端

  • AMS翻译模式（amsTranslationMode）
    切换Carpet AMS Addition的翻译模式

变化

  • 现在服务端默认使用客户端的语言向客户端发送翻译，你可以通过 AMS翻译模式（amsTranslationMode）规则来切换模式

  • 现在，在 AMS_NETWORK 分类下的规则在 AMS网络协议（amsNetworkProtocol）规则未启用时将无法开启，且当服务器启动时如果 AMS网络协议（amsNetworkProtocol）未开启，会将在 AMS_NETWORK 分类下的规则恢复至默认值

  • 现在，各别规则在设置它们时会将其强制设置为默认，且必须在重启服务器后生效，因此，大潜影盒（largeShulkerBox）规则已经变得足够安全

其他

  • 启用实验性矿车（experimentalMinecartEnabled）规则貌似有一点小问题，不过通过多重启几次服务器可以解决，不建议在自己的常玩存档中频繁切换该规则 #212

  • 修复了一些BUG（同时也有可能制造了新的BUG），具体是哪些懒得写了 XD

New Rules

  • preventServerPause
    Prevent server pause after 60s

  • flippinCactusExtras
    Enables flippinCactus rules to work on more blocks
    Affected blocks: barrel, chiseled_bookshelf, crafter, shelf

  • experimentalMinecartSpeed
    Set the speed of the experimental minecart
    Enabling this rule will override the game rules

  • experimentalMinecartEnabled
    Enable experimental minecarts
    When the server environment switches the rule setting, the server needs to be restarted
    When the client environment switches the rule settings, the client needs to be restarted

  • amsTranslationMode
    Switch the translation mode of Carpet AMS Addition.

Changes

  • The server now uses the client’s language by default when sending translated messages. You can switch this behavior using the amsTranslationMode rule.

  • Rules under the AMS_NETWORK category can no longer be enabled unless the amsNetworkProtocol rule is active. When the server starts with AMS Network Protocol disabled, all rules under the AMS_NETWORK category will automatically reset to their default values.

  • Some rules are now force-set to their default values when changed and require a server restart to take effect. As a result, the largeShulkerBox rule is now considered safe to use.

Other

  • The experimentalMinecartEnabled rule may behave inconsistently Restarting the server several times usually resolves the issue, but frequent toggling in your main world is not recommended.
    See #212 for details.

  • Fixed some bugs (and probably introduced new ones). Not going to list them all XD
